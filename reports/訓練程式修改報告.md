## 二、核心模組（`scripts/core/`）

### 1. `core/path_setup.py`（新增）

統一處理散落於各訓練/抓取腳本頂部、重複 2–4 次的 `sys.path.append` 區塊。

- 對外 API：
  - `ensure_scripts_on_path(caller_file=None)`：明確呼叫；可由任何位於 `scripts/` 子目錄
    的檔案傳 `__file__`，函式會從中推算 `scripts/` 根目錄並把所有子目錄
    （`fetchers / pipeline / training / monitor / models / utils / tests`）加入 `sys.path`。
  - **副作用 import**：模組被 `import` 時自動執行一次 `ensure_scripts_on_path()`，
    所以每支腳本只需一行 `import core.path_setup` 就能完成全部設定。
- 冪等：路徑已存在則跳過，可被多次 import 而不污染 `sys.path`。

### 2. `core/model_metadata.py`（新增，回應 P3-2）

提供完整的「訓練 → 封存 → 推論健檢」資料模型。

- `ModelMetadata`（dataclass）：欄位涵蓋
  - 基本：`stock_id / model_path / train_end_date / feature_count / feature_fingerprint`
  - 版本：`git_hash / python_version / package_versions / timestamp`
  - 績效：`oof_da / oof_sharpe / oof_ic / n_trades_per_fold / max_drawdown`
  - 訓練設定：`horizon_days / calibration_method / calibrator_cv / notes`
- `get_git_hash(short=True)`：取得 commit hash；失敗回 None（不阻斷訓練）。
- `fingerprint_features(features)`：對排序後特徵名稱算 sha256（前 12 hex），
  若推論時特徵集不一致可立即偵測。
- `save_metadata(metadata, archive_dir, also_archive_pkl=True)`：
  寫入 `outputs/models/archive/ensemble_<sid>_<timestamp>_<git>.metadata.json`，
  並可選擇將 `.pkl` 一併 `cp` 至 archive 目錄做歷史快照（供 Rollback）。
- `load_latest_metadata(stock_id, archive_dir)`：讀回最近一次 metadata。
- `assert_feature_schema_match(runtime_features, metadata, strict=False)`：
  在 predict 時呼叫，比對 fingerprint，不符合時視 strict 旗標警告或 raise。

### 3.3 [`parallel_train.py`](scripts/training/parallel_train.py)（rewrite）

- **INT-04 落實**：`concurrent.futures.ProcessPoolExecutor` → `multiprocessing.Pool.imap_unordered`
  - `ProcessPoolExecutor.submit()` 會立即實體化整個 future 列表；`imap_unordered` 採
    漸進式串流（worker 完成一個才送下一個 chunk），在標的數擴張至數百檔時可避免
    記憶體尖峰。
- **`KeyboardInterrupt` 優雅關閉**：`pool.terminate()` 後 `pool.join()` 確保子進程被回收。
- **Worker context**：`turbo / fast_mode` 改由 `Pool(initializer=_worker_init, initargs=...)`
  傳遞，取代原本檢查 `sys.argv` 的潛在競態。
- **保留**：`V3_REQUIRED_FEATURES` 守門（P0-5）、指數退避重試、health check 後處理、
  `aggregate_feature_importance_by_group()`、`get_stocks_needing_training()` 健康檢查整合。
- 新增 CLI flag：`--workers N` 取代之前硬編碼的 `MAX_WORKERS=6`，方便依硬體資源動態調整。

### 3.4 [`auto_train_manager.py`](scripts/training/auto_train_manager.py)（surgical edit）

- 僅修正 sys.path 重複插入。
- **保留**：心跳檔 + DB 雙通道、PID 檔追蹤、Tier-1/Tier-2 差異化重訓週期、
  指數退避、效能 priority sort、訓練前 feature store 預檢、SIXTH_WAVE_DRIVERS bonus
  等所有 v2 功能。

### 3.5 [`auto_predict_manager.py`](scripts/training/auto_predict_manager.py)（rewrite）

呼應檢核報告 P0-3「無告警機制」與 auto_train_manager 已具備的雙通道模式：

- **DB heartbeat**：新增 `auto_predict_heartbeat` 表（DDL 包含 PK + ts 索引），
  每次掃描週期都寫入；提供分散式監控 / 跨機器告警基礎。
- **檔案 heartbeat**：保留原有 `outputs/auto_predict.heartbeat`（本機 dashboard 用）。
- **PID 檔**：寫入 `outputs/auto_predict.pid`，配合 systemd / cron 監控。
- **訊號處理**：捕捉 SIGTERM/SIGINT，退出時清理 PID 檔並寫一筆 `STATUS=STOPPED` 心跳。
- **異常時心跳**：核心循環異常會寫入 `STATUS=ERROR` 心跳（含截斷的錯誤訊息），
  避免「靜默卡死」型故障。
- DB 不可用時所有 DB 相關呼叫 graceful fallback，不阻斷推論主流程。

### 3.9 [`tune_hyperparameters.py`](scripts/training/tune_hyperparameters.py)（rewrite）

### 3.11 [`feature_analysis.py`](scripts/training/feature_analysis.py)（rewrite）

呼應系統重構報告 ALG-01「滑動窗口計算與動能策略回測」加速建議：

- 新增 Numba `@njit(cache=True, fastmath=True)` 版的 rolling Pearson 相關係數
  `_rolling_pearson_numba(x, y, window)`：
  - 採 Welford 風格的累加變數（單次線性掃描，O(N·window)）。
  - 對 NaN 嚴格策略：窗口內任一 NaN → 該 timestep 輸出 NaN（與 pandas 行為對齊）。
- `_rolling_corr()` 共用包裝：依 `use_numba` 與 numba 安裝狀況自動切換。
- `FactorAnalyzer(..., use_numba=True)` 新增旗標：
  - `True` 且 numba 可用 → 啟用 JIT；典型加速 5–20×。
  - `True` 但 numba 缺席 → log info 後 graceful fallback。
- **保留**：IC / IC IR / decay 計算、`is_robust` 門檻（IC>0.02 & |IR|>=0.5）、
  `print_factor_report()` 輸出格式。
